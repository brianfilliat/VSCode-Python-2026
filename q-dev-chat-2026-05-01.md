Adding an HTTP MCP server
To add an HTTP MCP server to the IDE:

Access the MCP configuration UI.

Choose the plus (+) symbol.

Select the scope: global or local.

If you select global scope, the MCP server configuration is stored in ~/.aws/amazonq/default.json and available across all your projects. If you select local scope, the configuration is stored in .amazonq/default.json within your current project.

In the Name field, enter the name of the MCP server.

Select http as the transport protocol.

In the URL field, enter the URL that the MCP server will call when it initializes.

Under Headers - optional, you can enter key-value pairs that must be sent as HTTP request headers. .

Enter a Timeout value, as applicable.

Choose Save.

The documentation you shared doesn't specify a particular URL — the URL field is where you enter the endpoint of **your own MCP server**.

It depends on where your MCP server is running. Common examples:

- Local server: `http://localhost:3000/mcp`
- Remote server: `https://your-domain.com/mcp`

The URL is whatever endpoint your MCP server exposes to handle incoming requests. You need to know the address of the specific MCP server you're trying to connect to.

If you're looking for a specific public MCP server URL, you'd need to check that server's documentation directly.

MCP: server &#39;AWS&#39; failed to connect: SSE error: Invalid content type, expected &quot;text/event-stream&quot;

---


That error means the AWS MCP servers from that repo are **not HTTP/SSE servers** — they run as **local stdio processes**, not remote HTTP endpoints.

To fix this, you need to configure them as **stdio** transport, not HTTP:

1. In the MCP configuration UI, select **stdio** as the transport protocol (not http)
2. Set the **command** to run the server locally (e.g., `uvx` or `npx`)

Example for the AWS Documentation MCP server in `~/.aws/amazonq/default.json`:

```json
{
  "mcpServers": {
    "AWS": {
      "command": "uvx",
      "args": ["awslabs.aws-documentation-mcp-server@latest"],
      "env": {
        "FASTMCP_LOG_LEVEL": "ERROR"
      }
    }
  }
}
```

**Prerequisites:**
- Install `uv`: https://docs.astral.sh/uv/getting-started/installation/
- Or use `npx` for Node-based servers

Check the specific server's README in the repo for the exact command needed.
