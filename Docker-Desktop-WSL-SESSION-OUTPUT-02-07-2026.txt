ASSISTANT RESPONSE - WSL Path Explanation:
================================================================================

The path `\\wsl.localhost\docker-desktop\docker-desktop-user-distro` is a 
UNC (Universal Naming Convention) path for accessing WSL distributions from Windows.

Path Breakdown:
- \\wsl.localhost\        - Windows network prefix for WSL access
- docker-desktop\         - WSL distribution name (Docker Desktop's)
- docker-desktop-user-distro - Docker Desktop user distribution

Common Patterns:
- Docker Desktop: \\wsl.localhost\docker-desktop\docker-desktop-user-distro
- Ubuntu WSL: \\wsl.localhost\Ubuntu
- Generic: \\wsl.localhost\<distribution-name>

Command to list WSL distributions:
wsl --list --verbose

Key Notes:
- Requires WSL 2 (WSL 1 uses \\wsl$\)
- Distribution name is case-sensitive
- Distribution must be running for path access

### DOCKER DESKTOP WORKFLOWS

1. DOCKER DESKTOP WSL INTEGRATION
   - Docker Desktop uses WSL 2 backend
   - Creates docker-desktop and docker-desktop-data distributions
   - File access paths from Windows and WSL

2. FILE ACCESS PATTERNS
   - Docker Desktop: \\wsl.localhost\docker-desktop\docker-desktop-user-distro
   - WSL Ubuntu: \\wsl.localhost\Ubuntu
   - List distributions: wsl --list --verbose

3. COMMON WORKFLOWS
   Docker Commands:
   - docker --version
   - docker ps / docker ps -a
   - docker build -t myapp:latest .
   - docker run -d -p 8080:80 --name myapp myapp:latest
   - docker stop myapp
   - docker rm myapp

4. VOLUME MANAGEMENT
   - Create: docker volume create mydata
   - List: docker volume ls
   - Inspect: docker volume inspect mydata
   - Mount: docker run -v mydata:/app/data myapp:latest
   - Bind mount from Windows: 
     docker run -v /mnt/d/DOCU-2026/Python-vscode-2026:/workspace myapp:latest

5. DOCKER COMPOSE
   - Start: docker-compose up -d
   - Stop: docker-compose down
   - Logs: docker-compose logs -f
   - Rebuild: docker-compose up -d --build

6. DEVELOPMENT WORKFLOWS
   Python Container Example:
   docker run -it --rm \
     -v /mnt/d/DOCU-2026/Python-vscode-2026:/workspace \
     -w /workspace \
     python:3.11 \
     python your_script.py

   Interactive Shell:
   - Existing container: docker exec -it myapp bash
   - New container: docker run -it --rm python:3.11 bash

   VSCode Integration:
   - Dev Containers extension
   - Open Folder in Container feature

7. DATA LOCATIONS
   Windows:
   - %LOCALAPPDATA%\Docker\wsl
   - %APPDATA%\Docker

   WSL:
   - /var/lib/docker/volumes/
   - \\wsl.localhost\docker-desktop-data\data\docker\volumes\

8. MAINTENANCE COMMANDS
   - System cleanup: docker system prune -a
   - Disk usage: docker system df
   - Container cleanup: docker container prune
   - Image cleanup: docker image prune -a
   - Volume cleanup: docker volume prune

   File operations:
   - Container to host: docker cp myapp:/app/output.txt ./output.txt
   - Host to container: docker cp ./input.txt myapp:/app/input.txt

9. NETWORKING
   - Create network: docker network create mynetwork
   - Run on network: docker run --network mynetwork --name webapp nginx
   - Inspect network: docker network inspect mynetwork

10. TROUBLESHOOTING
    Docker Desktop won't start:
    - wsl --shutdown
    - Start Docker Desktop
    - wsl --update

    Check Docker daemon:
    - sudo service docker status
    - docker info

    Permission issues:
    - sudo usermod -aG docker $USER
    - Restart WSL

11. BEST PRACTICES
    - Use .dockerignore files
    - Implement multi-stage builds
    - Use specific image tags (avoid latest)
    - Add HEALTHCHECK to Dockerfiles
    - Set resource limits in Docker Desktop
    - Enable WSL integration in Docker Desktop settings

12. PATH CONVERSION
    Windows to WSL:
    D:\DOCU-2026\Python-vscode-2026 → /mnt/d/DOCU-2026/Python-vscode-2026

    WSL to Windows:
    /home/user/project → \\wsl.localhost\Ubuntu\home\user\project
